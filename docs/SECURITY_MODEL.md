# Security Model

## Authentication

Admin access is session-based. Login uses password hashes generated by `Auth::hashPassword()`, preferring Argon2id when available and falling back to PHP defaults.

Sessions are started through `SessionManager`, use activity enforcement, and regenerate session IDs after successful login.

## Roles And Permissions

The core full-access role is `admin`. “First admin” or “superadmin registration” refers to the one-time bootstrap flow, not to a separate persisted core role. Optional user-management modules can add restricted roles and granular permissions.

Examples:

- `edit_pages`
- `create_pages`
- `delete_pages`
- `view_versions`
- `restore_versions`
- `view_media`
- `manage_media`
- `create_blog`
- `edit_blog`
- `delete_blog`
- `view_analytics`
- `manage_redirects`
- `manage_settings`

## CSRF

State-changing admin APIs require POST and a valid CSRF token. Login, logout, first-admin setup, settings updates, media mutations, page/blog mutations, redirects, updates, health sync, and users API all use CSRF validation.

## Input And Output

- Stored public content is sanitized before rendering.
- Blog template partial paths are resolved through safe template resolution.
- Module manifests and promo metadata are sanitized before admin UI rendering.
- Public render errors should not expose internal paths to visitors.

## Uploads And Files

Media uploads are restricted by filename, extension, MIME checks, and path containment. Media filesystem operations should stay inside the uploads root.

Runtime data uses guarded files and must not be directly web-readable. The shipped Apache rules and documented Nginx baseline deny `config/`, `content/`, `data/`, `logs/`, `modules/`, `templates/`, backup/temp directories, and common runtime-file extensions. Keeping private runtime state outside the document root is preferred where supported.

## Integration API

Optional integration modules receive publishing webhooks through the Core endpoint at `/clipon/api/integrations.php`. Module directories remain private and must never be exposed as HTTP routes. The endpoint is sessionless, accepts JSON `POST` and `DELETE` requests, limits bodies to 5 MB, applies a per-provider/IP rate limit, and delegates authentication to the selected provider. Production integrations must use HTTPS.

Provider modules should store only token hashes, compare them in constant time, omit tokens and article bodies from logs, and rotate tokens from an authenticated, CSRF-protected administrator screen. Remote media importers must reject private/reserved network destinations and validate the downloaded file type.

## Analytics Privacy

Privacy/basic analytics avoids visitor cookies for anonymous public requests. Full analytics requires cookie banner acceptance and can use visitor/session state.
